1. Exercise 1: MICRO SERVO, IDE AND THE PHOTORESISTOR

This week, we were introduced to Arduino IDE which allow users to write code and upload it to any Arduino board. We also continued exploring the Arduino kit and looking at the different sensors available.

For the first exercise, we used a photoresistor to control the rotation of a micro servo motor. The IDE code enabled the servo to respond to changes in light levelsâ€”allowing users to influence its movement by approaching or moving away from the sensor.

Personally, I feel like we had a much easier time completing the circuit this time around compared to the first week. The challenge, however, was in applying the IDE code. Unlike traditional coding where it's entirely computer based, here multiple variables like the wiring of the circuit also have to be correct. That said, since this was a relatively simple exercise I think we went alright.

2. Exercise 2: ULTRASONIC

We were also working with an ultrasonic sensor which is often used to measure distance. In this exercise, we put the sensor into a circuit with a buzzer to then control its activation based on proximity recognition.

This exercise was one where the challenges of IDE application became apparent.

While we tried to set everything up according to the instructions, the process of getting everything to work was quite tedious. We spent a lot of time double checking everything and going over the diagrams before realising that the bug was due to a wiring error. It eventually worked though and the resulting human-sensor interaction was quite similar to that of the photoresistor in the first exercise except that this one was a lot more sensitive.

3. Exercise 3: POTATOES AND FOIL

For the last exercise, we turned individual wires into touch-sensitive inputs that each triggered a different sound when touched using an extension library.

Completing the circuit was alright, but like the second exercise, my group once again had a hard time with debugging and making sure that the code work. We never ended up being able to get one of the wires to respond but 2/3 of them did so I still counted that as a win.

The second photo is technically inaccurate as we were supposed to only stimulate one wire at a time but piercing the potato was fun.

Ultimately, I came to the conclusion that the more interesting and complex an Adruino's sensor logic is, the more difficult it is to both set up and maintain. Therefore, it might be more efficient for me as a beginner (if i do physical computing for my final project) to set up a bunch of simple commands, rather than trying to create a really complex condition with advanced inputs and outputs.
