i. week 5's homepage

I started the week off by researching different p5.js examples and snippets. Through my digging, I was able to find p5.3D, an extension library that allows users to easily generate and render 3D assets in p5.js using functions like Word3D, Picture3D and Canvas3D.

I was quite intrigued by this library as it renders the 3D assets in a y2k-esque way that aligns with the aesthetic that I'm trying to pursue. Using what I've learnt so far, I began testing out different animation effects.

Ultimately, I found the idea of the 3D letter trail to be the most interesting and wanted to incorporate it into my week 5 workbook. In order to optimise this sketch for interaction, I added a keyPressed function that allows the user to clear the canvas upon clicking any keyboard character. This is to give the user the flexibility to interact with the sketch for as long as they want without worrying about reaching a 'stopping' point when the texts begin to overcrowd. I tried to avoid using any 'click' or 'drag' movements unnecessarily as the library rendering naturally allow users to pan around the canvas on drag.

I also enforced a character limit on the text trail as 1) it prevents the trail from lagging once it gets too long and 2) looks kind of cool.

I did a few sketches to help visualise what the composition of the page was going to look like. I was a fan of the juxtaposition between the busy, chrome-coloured typography and a very plain white background, so I was looking for ways to enforce a type of 'boundary' that may help to highlight this contrast.

After playing around with different combinations of objects and visual styles, I settled into an exaggeratedly stock image type aesthetic. To add another layer of interaction onto the page, I made all of the images clickable and draggable assets that link to my inspirations for this week. Dragging the items around may also help the user to recognise that the central MS Paint canvas can be played with even without the help of the instruction scroll.
